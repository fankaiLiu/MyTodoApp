## todoManager——服务端
个人毕业设计仓库
论文题目：
基于 Rust语言 的内存安全高性能 To-Do 系统的设计与实现
### 项目介绍
前后端分离的web端任务管理系统
具有离线、在线两种模式，支持个人离线使用(不联网)与团队在线协作
### 架构设计
采用前后端分离架构，后端使用 Rust 语言及 Salvo 框架开发 RESTful API；
前端采用 Rust 编写的 Leptos 框架实现同构 Web 界面；
数据库选用 PostgreSQL 或 SQLite 存储任务数据；
通过 JSON/MsgPack 完成前后端通信。
### 项目技术栈
- 前端：Leptos(SSR模式)
- 后端：salvo
- 数据库：SQLx(SQLite) 
- 通信协议：quic协议
- 异步运行时：tokio
- 网络协议：ipv6
- 传输协议：WebTransport
### 项目功能
- 明暗主题切换
- 单机模式
    - 如果是单机模式，用户注册与登录后，所有任务都在本地存储，不与服务器交互
- 用户模块
    - 用户注册与登录
    - 用户信息管理(注册时间、登录时间、邮箱、手机号、密码、所加入的团队等)
- 任务模块
    - 子功能: 任务收藏
    - 任务的增删改查
    - 任务状态管理(待办、进行中、已完成)、任务优先级管理(u8类型)
    - 任务的struct定义(任务id、名称、状态、优先级、创建时间、更新时间、完成时间、任务类型(个人或团队)、任务创建者(用户id)、任务所属团队(团队id))
    - 任务属性: 除了上面的基本属性外，包括(任务难度)
    - 子任务: 可以为每个任务创建子任务，子任务的状态会受到父任务状态的影响
- 团队模块
    - 团队的创建与加入
    - 团队成员管理(添加、移除、查看成员)
    - 团队任务管理(创建、分配、查看、完成)
- dashboard模块
    - 个人任务展示(待办、进行中、已完成)
    - 团队任务展示(待办、进行中、已完成)
    - 任务详情展示(任务名称、状态、优先级、创建时间、更新时间、完成时间、任务类型、任务创建者、任务所属团队)
- 日志模块
    - 记录用户及团队操作日志(登录、注册、创建任务、分配任务、完成任务、加入团队、退出团队等)
    - 记录管理人员操作日志(创建团队、删除团队、添加团队成员、移除团队成员等)
